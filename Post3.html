“Vanilla” Javascript Vs. JQuery

Two annoy languages

Posted by Jackson Holliday on October 9, 2020

Years ago websites used to be bland and static like. However, “vanilla” JavaScript came around in 1995 and revolutionized the way sites looked and function. The actual term “Vanilla” Javascript didnt get coined till 2012 tho. Basically in short, it helped developers create current, more modern sites look and feel far more dynamic instead of it being just bland and static. Then JQuery came around in 2006.

JQuery is a library of tools created by developers around the world, using Javascript. In simple words, jQuery is a lightweight, easy to use JavaScript library that helps in creating complex functionalities with few lines of coding. JQuery coding is shorter and often more simplified than equivalent vanilla JS code. It's a library that has a high level of abstraction, making it easy to use but leaving out various learning opportunities in the process. Importantly, it’s incredibly flexible within the browser, meaning that it’s compatible with every browser available on the market (Firefox, Chrome, Edge, Opera). Developers will not even need to worry about the browser that the user may be using at the time.

JavaScript is a dynamic computer programming language. It’s lightweight and commonly used as a part of web pages, whose implementations allow client-side script to interact with the user and create dynamic pages. It is an interpreted programming language with object-oriented capabilities.

JavaScript

  • Weakly typed dynamic programming language
  • Used for creating dynamic websites user friendly interfaces
  • All the coding must be done from scratch. Might turn out to be difficult and time consuming for novice programmers
  • Developers have to take care of multi browser compatibility. Browser related errors are more like to appear
  • No additional plugins are required to run JS. Do not need to add any additional plugins as all browsers support JavaScript
  • Long line of code. May lead to spaghetti codes
  • DOM can be accessed faster
  • Not easy to learn. Learning curve is steep

jQuery

  • Rich, lightweight, and easy to use JavaScript library
  • JS framework used to handle AJAX requests, manipulate CSS, and create animations like as slide, hide etc
  • Has several pre-written functions. Only the necessary modifications has to be made. Time saver
  • Works with all modern browsers. There is no such compatibility issue with any browser.
  • Library script link has to be included inside the head tag of the webpage.
  • Less coding is required to do the same work.
  • Best for complex operations where developers usually make mistakes and write poor lines of code
  • Far easier to learn than JS
